# Screenshots Guide

This document provides guidance on preparing screenshots for the project.

## Required Screenshots

Please capture the following screenshots for the README and Product Hunt:

### 1. Main Interface (Trace Page)
- **File**: `docs/screenshots/trace-page.png`
- **Description**: Show the activity timeline with AI summaries
- **Key Elements**: Timeline, activity cards, tags, footer statistics

### 2. Statistics Dashboard
- **File**: `docs/screenshots/statistics-page.png`
- **Description**: Show the statistics page with charts and metrics
- **Key Elements**: Stat cards, API performance, token distribution

### 3. Summary Page
- **File**: `docs/screenshots/summary-page.png`
- **Description**: Show the daily summary with charts
- **Key Elements**: Daily summary card, comparison with yesterday, weekly/monthly/yearly charts

### 4. Settings Page
- **File**: `docs/screenshots/settings-page.png`
- **Description**: Show the settings page with all configuration options
- **Key Elements**: API key, model selection, prompts, resolution settings

### 5. Multi-language Demo
- **File**: `docs/screenshots/multilang-demo.png`
- **Description**: Show the app in both English and Chinese
- **Key Elements**: Language switcher, localized content

## Screenshot Guidelines

1. **Resolution**: Use high-resolution screenshots (at least 1920x1080)
2. **Format**: PNG format preferred
3. **Content**: Use realistic data (not placeholder text)
4. **Privacy**: Ensure no sensitive information is visible
5. **Consistency**: Use the same theme/style across all screenshots

## Product Hunt Specific

For Product Hunt, prepare:
- **Hero Image**: 1280x720px (main product image)
- **Gallery Images**: 5-8 additional screenshots
- **Video**: 30-60 second demo video (optional but recommended)

## Adding Screenshots

1. Place screenshots in `docs/screenshots/` directory
2. Update README.md to reference the screenshots
3. Ensure screenshots are optimized for web (consider compression)
